# Инструкция по установке и запуску

## 1. Установка Node.js

Для работы с веб-приложением необходима установка Node.js версии 18 или выше.

### Windows

1. Скачайте установщик Node.js с официального сайта: https://nodejs.org/ru/download/
2. Выберите версию "LTS" (Long-Term Support)
3. Запустите установщик и следуйте инструкциям
4. После установки перезапустите командную строку или PowerShell
5. Проверьте версию, выполнив команды:
   ```
   node -v
   npm -v
   ```

### Linux (Ubuntu/Debian)

```bash
# Установка через NVM (рекомендуется)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.3/install.sh | bash
source ~/.bashrc
nvm install --lts
nvm use --lts

# Или через apt (может установить устаревшую версию)
sudo apt update
sudo apt install nodejs npm
```

## 2. Установка зависимостей проекта

После установки Node.js:

```bash
# Перейти в каталог web
cd web

# Установить зависимости
npm install
```

## 3. Запуск в режиме разработки

```bash
# Запустить сервер разработки
npm run dev
```

Приложение будет доступно по адресу http://localhost:5173

## 4. Решение проблем

### Проблема с запуском Vite

Если при запуске `npm run dev` возникает ошибка с текстом:
```
await import('source-map-support').then((r) => r.default.install())
^^^^^
SyntaxError: Unexpected reserved word
```

Это означает, что у вас устаревшая версия Node.js. 

**Решение:** Обновите Node.js минимум до версии 16.0.0 (лучше до 18.x или 20.x LTS).

### Проблемы с установкой зависимостей

Если при установке пакетов возникают ошибки:

1. Очистите кэш npm:
   ```
   npm cache clean --force
   ```

2. Удалите папку node_modules и повторите установку:
   ```
   rm -rf node_modules
   npm install
   ```

### Проблемы при сборке

При возникновении ошибок в процессе сборки:

1. Проверьте наличие всех необходимых файлов в проекте
2. Убедитесь в отсутствии ошибок TypeScript:
   ```
   npm run lint
   ```

## 5. Настройка API Gateway

Для полноценной работы приложения необходим запущенный API Gateway на порту 8080.

Если API Gateway работает по другому адресу, измените настройки в файле `vite.config.ts`.
